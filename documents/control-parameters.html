control parameters

The are a number of pre-defined JavaScript variable names that define parameters that may be used to control the way that Thymol behaves when running, these parameters have names the beginning with the prefix "th".

scalar valued parameters

  • thRoot & thPath
  • thRelativeRootPath
  • thProtocol
  • thDebug
  • thMessagePath
  • thMessagesBaseName
  • thLocalMessages
  • thDisableMessages
  • thLocale
  • thPrefix
  • thDataPrefix
  • thExtendedMapping
  • thIndexFile
  • thTemplateSuffix
  • thDefaultPrecedence
  • thDefaultPrecision
  • thAllowNullText
  • thShowNullOperands has been deprecated

You may set any of the scalar valued parameters by one of the following mechanisms:

  • Allow them to default (note that for some of these parameters, the default value may by reconfigured).
  • Set them using a script tag in the template source file.
  • Set them in a JavaScript source file imported into your template using a script tag.
  • Set them as query parameters on the referring link to the template.
  • Set them as parameters using the "data-thymol-parameters" attribute of the script tag that defines the Thymol import.

e.g. The value of thProtocol may be set to "http://" for a particular template using:

      <script type="text/javascript" src="../../../js/thymol.js" data-thymol-parameters="?thProtocol=http://"></script>

structure valued parameters

  • thVars
  • thMessages
  • thMappings
  • thDisable

A detailed description of the effects of each of these parameters is discussed in further sub-sections of this page. You can reach these using the sub-menu selections on the left.

Setting the values of the structure valued parameters is best done in JavaScript.
